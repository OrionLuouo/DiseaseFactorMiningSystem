# XGBoostClassifierChain 多标签疾病预测模型文档

> 基于 `data_stage02.csv`，针对 19 个疾病标签的极端稀疏多标签分类任务。

---

## 一、数据概况

| 属性 | 值 |
|------|------|
| 数据来源 | `data/data_stage02.csv`（由 stage02 预处理生成） |
| 总样本数 | 6961 |
| 特征数量 | 44 |
| 标签数量 | 19 |
| 标签稀疏度 | ~95% 为零（极端不平衡） |
| 最稀有病 | 脂肪肝/骨质疏松/乙肝，正样本仅 11-12 例 |
| 最密有病 | 蛀牙，正样本 292 例 |
| 训练/测试划分 | 80% / 20%，按"患病数量"分层 |

---

## 二、模型搭建思路

### 2.1 核心问题

- 19 个标签中仅 4.7% 是非零元素（95.3% 为零）。
- 最稀有病（乙肝）正样本占比仅 0.39%，最密（蛀牙）仅 15.64%。
- 传统 `accuracy` / Top-k 在这种场景下评估分数虚高但无实际意义。

### 2.2 模型架构：XGBoostClassifierChain

```
输入特征 X (n_samples × 44)
  │
  ├─ 标签 0: XGBoost 分类器 → 概率 p₀
  ├─ 标签 1: XGBoost 分类器，输入 [X, p₀] → 概率 p₁
  ├─ 标签 2: XGBoost 分类器，输入 [X, p₀, p₁] → 概率 p₂
  └─ ... 以此类推（链式传递）
```

**设计理由：**

- **ClassifierChain（链式分类）**：将前 j-1 个标签的预测概率作为第 j 个标签的额外特征，利用标签间相关性（如"高血压"和"糖尿病"常共病）。
- **XGBoost 基分类器**：梯度提升树天然支持不平衡数据（`scale_pos_weight`），对极端稀疏标签的表现远优于线性模型。
- **逐标签独立权重**：每个标签使用独立的 `scale_pos_weight = n_neg / n_pos`，稀有标签自动获得更高权重。

### 2.3 关键参数

| 参数 | 值 | 说明 |
|------|------|------|
| `max_depth` | 6 | 树深度，兼顾拟合与泛化 |
| `learning_rate` | 0.1 | 学习率 |
| `n_estimators` | 200 | 树的数量（CV 阶段为 150） |
| `subsample` | 0.8 | 行采样比例 |
| `colsample_bytree` | 0.8 | 列采样比例 |
| `reg_alpha` | 0.1 | L1 正则化 |
| `reg_lambda` | 1.0 | L2 正则化 |

### 2.4 训练流程

```
1. 加载数据（data_stage02.csv）
2. 分层划分训练/测试集（按患病数量分层：0=健康, 1=单病, 2+=多病）
3. 计算 per-label scale_pos_weight
4. 5 折分层交叉验证（StratifiedKFold），选取 F1-macro 均值
5. 全训练集训练最终模型
6. 在验证子集（训练集 10%）上搜索最优决策阈值
7. 测试集全面评估
8. 保存模型（joblib）+ 评估报告（JSON）
```

## 三、评估思路

### 3.1 为什么不用传统 accuracy？

多标签分类中，如果标签稀疏（95% 为零），模型只需全预测为 0 就能获得 95% 的 accuracy，但这完全无意义。因此需要更精细的评估体系。

### 3.2 9 个全局评估指标

| 指标 | 说明 | 适用场景 |
|------|------|----------|
| **Subset Accuracy** | 所有标签全对的样本比例 | 最严格，要求一个不差 |
| **Hamming Loss** | 平均每个标签的误判率 | 反映单标签错误率 |
| **F1-macro** | 各标签 F1 等权平均 | **最常用**，不受常见病主导 |
| **F1-micro** | 全局 TP/FP/FN 统一计算 | 反映整体预测能力 |
| **Precision-macro** | 精确率等权平均 | 衡量预测的可靠性 |
| **Recall-macro** | 召回率等权平均 | 衡量覆盖能力 |
| **AUC-ROC (macro)** | 宏观 AUC | 对不平衡数据较鲁棒 |
| **AUC-PR (macro)** | 宏观平均精度 | **更适合极端不平衡**的场景 |
| **Label Ranking AP** | 标签排序平均精度 | 衡量多标签排序能力 |

### 3.3 逐标签详细指标

每个疾病独立计算：

| 指标 | 说明 |
|------|------|
| F1 | 精确率与召回率的调和平均 |
| Precision | 预测为正的样本中实际为正的比例 |
| Recall | 实际为正的样本中被预测正确的比例 |
| Support | 测试集中实际的正样本数量 |
| Threshold | 为该标签搜索的最优决策阈值（替代默认 0.5） |

### 3.4 阈值优化策略

- **默认阈值 0.5** 在不平衡数据中效果不佳。
- 在验证集上遍历 `[0.05, 0.95]`（步长 0.05），选取使 F1 最大的阈值。
- 每个标签使用独立的最优阈值。

### 4.5 结论

**原始数据（baseline）优势：**
- 高血压、低血压、睡眠呼吸暂停、心律失常、贫血、NAFLD 这 6 类常见病预测效果优秀（F1 > 0.85）。
- AUC-ROC = 0.8066 表明模型整体具有较好的区分能力。
- 阈值优化有效降低了误判率（Hamming Loss = 0.0368）。

**原始数据（baseline）不足：**
- **10 个稀有病完全预测不出来（F1=0）**，根本原因是训练样本太少（≤15 例）。
- F1-macro = 0.3685 受稀有病严重拖累。
- 感冒/蛀牙/糖尿病处于中间地带，特征与其他病重叠或样本不足。

**增强后效果（数据增强前 vs 后）：**
- F1-macro 从 **0.3685 → 0.7260**（+97% 提升）
- AUC-ROC 从 **0.8066 → 0.9187**（+14%）
- AUC-PR 从 **0.4068 → 0.7578**（+86%）
- Subset Accuracy 从 **0.5179 → 0.6435**（+24%）
- **新增 12 个疾病 F1 > 0.8**，其中原来 F1=0 的 6 个病（脂肪肝、骨质疏松、肾衰竭、乙肝、痛风、胆结石）全部进入优秀行列
- 仍存在 3 个病（蛀牙、感冒、糖尿病）效果不佳，需要补充专属特征或采用其他方法

**下一步改进方向：**
1. 对蛀牙/感冒/糖尿病，考虑补充专属特征或引入诊断规则辅助预测。
2. 尝试 SMOTE 上采样进一步提升稀有病效果。
3. 尝试不同的链式标签排序策略（当前为随机排序）。
4. 对丙肝/肾结石/肠胃炎继续增加合成数据样本量。
