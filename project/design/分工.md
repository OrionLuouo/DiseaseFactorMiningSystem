# <center> 分工

## 1. 工作流程细化
  - ***确定数据类型及来源***：例如，确定“体检数据”分为“身高”、“体重”、“血压”三维。同时，确定数据的形式，如病症以“感冒”、“胃炎”、“哮喘”等值作分类，睡眠时间以浮点数作回归。再确定数据来源，从哪些网站、数据库，怎样获取。
  - ***数据采集***：编写程序从已确定途径自动化获取数据。
  - ***数据清洗***：编写程序将采集到本地的数据进行预处理，去除或修正异常、缺失数据。由于我们不采用自然语言处理和符号系统，所以针对分类型的特征，需要人工编制字典，例如将“头疼”“头痛”“脑袋疼”“脑袋痛”等常见描述统一化。
  - ***数据处理***：编写程序将数据作额外处理，转换为适用于机器学习的格式与形式。
  - ***模型训练***：编写训练程序，使用处理好的数据训练用于疾病预测、影响度分析、建议生成的模型。需要具备两种模式：1.正向推断结果，根据可能的症状 + 用户状况，推断潜在疾病风险。2.反向推断原因，根据患者已有疾病 + 用户状况，推断发病原因，并给出改善建议。
  - ***预测模型***：编写预测程序，使用已训练的模型，对新输入的个体数据进行处理。需要人工确定推断阈值，例如超过 xx% 概率患病时做出提醒、发病原因影响权重超过 xx% 时标记为病因。
  - ***Web 服务器***：编写后端程序，搭建 Web 服务器，响应用户请求。
  - ***前端页面***：编写前端页面，实现输入、请求、响应等功能，呈现最终的可视化报告。

## 2. 具体要求
  - #### 在开发初期，由于多阶段并行，一些事项无法精确对接，例如数据的特征维度、Web 服务器响应格式等，需要在对接处做保留性设计，后期再通过协商确定。这些内容包括：
    - *数据采集端-模型训练端*：可以预先将特征数据分为离散特征与线性特征，并不确定数量、名称等等，模型训练程序设定动态的特征维度，通过全局变量等方式进行统一管理。数据采集、清洗完毕后，将数据连同标准一同发送给模型训练端。
    - *Web 服务器-前端页面*：初期由于模型训练和预测模型需要同步开发，Web 服务器开发端暂无法得知预测返回的结果类型等，无法做响应逻辑，可以只先做用户信息采集部分。或者 Web 服务器开发端与前端页面开发端协商好固定的报告响应格式，最后由 Web 服务器开发端做预测结果到报告响应格式的适配。
    - *数据采集端-前端页面*：采集用户信息时，需要根据系统已考虑特征来确定，哪些特征是可选的，哪些是必选的，以及可选特征的适宜默认值是什么。当然，这个也可以做成动态的，即前端数据采集完毕后，将已考虑特征、是否必选、默认值等信息入库，后端渲染页面时再向 Web 服务器请求所需特征有哪些，根据响应内容动态渲染。
