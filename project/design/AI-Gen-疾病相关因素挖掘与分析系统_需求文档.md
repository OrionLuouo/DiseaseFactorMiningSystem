# 基于关联规则的疾病相关因素挖掘与分析系统

## 一、项目概述

### 1.1 项目背景

疾病的发生是多种因素共同作用的结果，包括生活习惯、遗传、环境、职业暴露等。目前医疗机构积累了海量的临床病例数据，但这些数据中蕴含的潜在关联关系尚未被充分挖掘。本项目旨在构建一个**疾病相关因素智能挖掘与分析平台**，运用关联规则算法，从多维度数据中提取疾病与危险因素之间的潜在关联，为疾病预防、病因研究和个性化诊疗提供数据支撑。

### 1.2 项目目标

- 整合多源异构数据（临床病例、体检、生活习惯、环境、基因）
- 运用关联规则算法挖掘疾病与危险因素的潜在关联
- 生成可视化分析报告与关联图谱
- 为疾病预防、病因研究、个性化诊疗提供决策依据

### 1.3 适用范围

面向医疗机构、科研院所、公共卫生部门。

## 三、工作流程

### 3.1 整体流程概览

```
数据源采集 → 数据清洗 → 特征工程 → 关联规则挖掘 → 模型构建 → 可视化 → 报告输出
```

### 3.2 详细工作流程

#### 阶段一：数据采集

```
数据源确认 → 数据抽取 → 格式转换 → 数据加载
```

**数据来源明细：**

| 数据类别 | 来源系统 | 数据格式 | 采集方式 |
|----------|----------|----------|----------|
| 临床病例数据 | HIS（医院信息系统） | 结构化表格 | ETL 定时抽取 / API 接口 |
| 体检数据 | 体检管理系统 | 结构化 + 半结构化 | ETL 定时抽取 |
| 生活习惯数据 | 问卷调查系统 / App | 结构化 | API 接口 / 文件导入 |
| 环境数据 | 环保部门公开数据 / IoT 传感器 | 结构化 / 时序数据 | API 爬取 / 消息队列 |
| 基因数据 | 基因检测平台 | 半结构化 (VCF) | 文件导入 |

#### 阶段二：数据预处理

```
数据加载 → 数据清洗 → 缺失值处理 → 异常值检测 → 数据标准化
```

**预处理步骤：**

| 步骤 | 说明 | 处理方法 |
|------|------|----------|
| 去重 | 删除重复记录 | 基于主键去重 |
| 缺失值处理 | 对缺失字段进行补全或删除 | 均值/中位数/众数填充，或剔除缺失率 > 30% 的样本 |
| 异常值处理 | 识别并处理离群点 | 3σ 原则 / IQR 四分位法 |
| 数据标准化 | 统一量纲 | Min-Max 归一化 / Z-Score 标准化 |
| 数据离散化 | 连续值转离散值（为关联规则做准备） | 等频分箱 / 等距分箱 |

#### 阶段三：特征工程

```
特征选择 → 特征编码 → 特征构造
```

**特征处理方式：**

| 特征类型 | 处理方式 | 示例 |
|----------|----------|------|
| 数值型特征 | 离散化分箱 | 年龄 → 青年/中年/老年 |
| 类别型特征 | One-Hot / Label Encoding | 性别 → 男=1, 女=0 |
| 文本型特征 | 关键词提取 / 嵌入 | 症状描述 → 标准化症状词 |
| 时序特征 | 滑动窗口统计 | 近1年体检指标变化趋势 |

#### 阶段四：关联规则挖掘

```
数据转换（布尔矩阵） → Apriori/FP-Growth 算法执行 → 规则筛选 → 规则评估
```

**关联规则评估指标：**

| 指标 | 公式 | 说明 |
|------|------|------|
| 支持度 (Support) | P(A ∩ B) | 同时出现 A 和 B 的概率 |
| 置信度 (Confidence) | P(B\|A) | 出现 A 时出现 B 的条件概率 |
| 提升度 (Lift) | P(B\|A) / P(B) | 大于 1 表示正相关，小于 1 表示负相关 |

**规则筛选标准：**
- 最小支持度：≥ 0.01（根据数据规模调整）
- 最小置信度：≥ 0.6
- 最小提升度：≥ 1.5

#### 阶段五：聚类分析

```
特征选择 → 距离度量 → 聚类执行 → 结果评估
```

- 算法：K-Means / DBSCAN
- 评估指标：轮廓系数 (Silhouette Score)、Calinski-Harabasz 指数

#### 阶段六：模型构建与预测

```
训练集/测试集划分 → 特征选择 → 模型训练 → 模型评估 → 模型部署
```

#### 阶段七：可视化与报告

```
数据展示 → 关联图谱生成 → 趋势分析 → 报告导出
```

## 四、数据来源与处理细节

### 4.1 数据来源

| 数据类别 | 具体字段 | 来源系统 |
|----------|----------|----------|
| 患者基本信息 | 年龄、性别、职业、居住地 | HIS 系统 |
| 临床诊断数据 | 疾病名称、ICD-10 编码、确诊时间 | 电子病历系统 |
| 体检数据 | 血压、血糖、血脂、BMI、肝功能等 | 体检管理系统 |
| 生活习惯数据 | 吸烟史、饮酒史、饮食频率、运动习惯 | 问卷调查 |
| 环境数据 | PM2.5、水质等级、职业暴露史 | 环保部门 API |
| 基因数据 | 基因突变位点、家族病史 | 基因检测平台 |

### 4.2 数据处理流程

```
原始数据 → 数据加载 → 数据清洗 → 特征工程 → 关联规则输入格式
```

**具体处理：**

1. **数据清洗**：
   - 去除重复记录（`drop_duplicates`）
   - 剔除关键字段缺失的样本（`dropna`）
   - 去除格式错误的数据（如日期格式不规范）

2. **数据离散化（关键步骤）**：
   - 连续型数值 → 分类区间（用于关联规则输入）
   - 例：年龄 18-30 → "青年"，BMI < 18.5 → "偏瘦"，18.5-24 → "正常"

3. **数据编码**：
   - One-Hot 编码：将类别特征转换为布尔矩阵
   - 示例：疾病"高血压" → 列 `has_hypertension`，有=1，无=0

### 4.3 如何量化

| 原始特征 | 量化方式 | 量化结果 |
|----------|----------|----------|
| 年龄 | 分箱 | 青年/中年/老年 |
| 血压 | 分箱 | 正常/偏高/高血压 |
| 血糖 | 分箱 | 正常/偏高/糖尿病 |
| BMI | 分箱 | 偏瘦/正常/超重/肥胖 |
| 吸烟史 | 编码 | 从不/偶尔/经常 |
| 职业 | 编码 | 脑力劳动/体力劳动/有毒有害 |
| 环境质量 | 编码 | 优/良/轻度污染/重度污染 |

### 4.4 最终分类

关联规则挖掘完成后，疾病相关因素按以下维度分类：

| 分类维度 | 分类结果 |
|----------|----------|
| 危险因素类型 | 可改变因素（饮食、运动、吸烟）/ 不可改变因素（年龄、性别、遗传） |
| 关联强度 | 强关联（Lift > 3）/ 中等关联（1.5 < Lift ≤ 3）/ 弱关联（1 < Lift ≤ 1.5） |
| 影响方向 | 正向关联（增加疾病风险）/ 负向关联（降低疾病风险） |
| 疾病类别 | 心血管疾病 / 代谢性疾病 / 呼吸系统疾病 / 肿瘤等 |
